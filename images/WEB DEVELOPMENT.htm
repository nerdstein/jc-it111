HTML and Beyond

 

HyperText Markup Language (HTML) is the language of the internet, more specifically, it is a hypermedia document description language that is used to publish documents on the World Wide Web.  HTML files are in an ASCII format, meaning that you can use any old text editor to create an HTML file.  Simply use a text editor such as Notepad to create or edit HTML.  Save the file with an HTML file extension.  Then, examine the file with a browser such as Internet Explorer or Netscape.  Transfer the file to your web space and then use the browser to examine the page on its web space. 

 

Here is a simple HTML file:

 

<html>

<head>

<title>

This is a Simple HTML Page.

</title

<h1>

This is what h1 looks like.

</h1>

</head>

<body>

The body is the section where the text and graphics appears.  This is a simple page so we’ll start with text only. 

<p>

Here we will place a new paragraph.

</body>

</html>

 

If you haven’t written basic HTML files before, simply copy the above into an editor and save it as an HTML file.  Then open it in MS Internet Explorer.

 

An HTML file is composed of elements which represent various structures (such as a header), or a desired behavior (such as italic).  Most elements include a starting tag, content, and an ending tag. 

 

HTML markup tags consist of a left bracket (<), followed by the tag name, and then a right brackt (>).  Tags usually are in pairs whereby the first tag indicates the start of an element and the second tag preceded by a slash (/) indicates the end.  For example,

<H1>A Sample Web Page</H1>

There are also certain HTML elements that do not hold any content.  These elements do not have any ending tags.  Examples include a horizontal rule (<HR>) and a line break (<BR>).

 

Certain browsers may interpret certain commands differently or may not recognize certain commands.  For now, we’ll cover HTML coding for most web browsers.  So, here are a few universal HTML commands.

 

-  HTML identification <html> </html> identifies the beginning and end of your HTML

code.

-  Header name <head> provides the header information for the page.

-  Title page <title> </title> specifies the title to be placed at the top of the browser window when the page appears.

-  Headings <h1> </h1>, <h2> </h2> etc through <h6> </h6> (capitalization is irrelevant)

-  Paragraphs <p> </p>

-  Horizontal Rules <hr>

-  Line Breaks <br>

-  Basic Text Styles <b> </b> for bold; <u> </u> for underline, <I> </I> for italic and

<em> </em> for emphasize

 

When you have more than one set of tags together, they must follow nesting rules.  In general, nesting rules tend to follow grammatical rules for brackets.  For example,

{( xyz)}is correct but {( }) is incorrect.  Likewise, <b> <i>This is bold and italic</i> </b> is correct but ending with the sequence </b> </i> is incorrect.

ENTER JAVASCRIPT

 

Now, to make your page more dynamic and interesting, enter JavaScript!

 

The most popular reason for using JavaScript is “the rollover”.  This feature allows the visitor to your web site to view different images depending on whether or not their mouse is rolled over a particular image or text in your web site.  An example of rollover code follows:

 

<HTML>

<HEAD>

            <TITLE>Link Rollover</TITLE>

<script language="JavaScript">

            <!--Hide script from old browsers

 

if (document.images) {

    final = new Image

    finhov = new Image

 

            final.src = "final.gif"

            finhov.src = "final-hover.gif"

}

else  {

            final = ""

            finhov = ""

            document.fin = ""

}

 

// End hiding script from old browsers -->

</SCRIPT>

</HEAD>

<BODY BBGCOLOR="CORNFLOWER">

<h1><A HREF="next.html" onMouseover="document.fin.src=final.src"

onMouseout="document.fin.src=finhov.src">

<IMG SCR=images/final-hover.gif" WIDTH=147" HEIGHT="82" NAME="fin" ALT="fin"></A>

</BODY>

</HTML>

 

In order for this to work, you need to make certain that your .gif files are available in the same directory as your program file.

if (goodEmail){

   good = true

} else {

   alert('Please enter a valid e-mail address.')

document.TheForm.focus() // put the cursor in the email field again

   field.focus()

   }

}

 

function sendOff(){

      alert('Please enter your name.')

        document.TheForm.emx.value)&&(good)){

Regardless of what you use to develop your web page, there are certain design issues that you need to take into consideration.  User-Centered Web Development by Jonathan Lazar (2001) is a text that addresses such concerns.  According to Lazar, there are seven stages to the user centered web development cycle.  The first involves defining who will use your web site and what are the goals of your site.  In other words, are you hoping to sell a product or service?  Do you merely wish to provide information on upcoming events or a possible drug interaction or side effect? 

 

The second stage involves collecting the user requirements for the web site.  This involves determining characteristics of the user such as what type of connection they use and what they expect to see out of a web site. 

 

Stage three involves creating the conceptual design of the web site.  The following questions should be asked and answered: “What page layout should be selected?”  “What graphics will be used.”  “What colors should be used?”    The following URL addresses specific issues regarding color selection and the effects of color blindness on site appearance:

 http://www.firelily.com/opinions/color.html

Stage four deals with the physical design of the site.  This involves the actual coding of the site.  While stage five involves site testing.  Stage six is implementation and marketing of the site.  Stage seven involves evaluating and improving the site.

So, you see that web development is not simply a matter of throwing together a web page.  However, in order to get comfortable with publishing web sites, it is important to put something out there!  So, don’t be afraid of mistakes and don’t wait for the perfect site, get out there and put your site on the web!

References:

Lazar, Jonathan, 2001. User-Centered Web Development, Jones and Bartlett Publishers, Sudbury, MA.
